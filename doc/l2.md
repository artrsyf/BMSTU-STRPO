# Отчёт о выполнении ЛР№2

## Наименование работы

Разработка и детализация функциональных требований.

## Детализация требований в виде пользовательских историй

### Student Context
- Stu-01-001 : Я, как администратор автошколы, хочу зарегистрировать ученика, чтобы начать процесс его обучения
- Stu-01-002 : Я, как администратор автошколы, хочу просматривать список зарегистрированных учеников, чтобы контролировать набор в автошколу
- Stu-01-003 : Я, как ученик, хочу просматривать свой текущий статус обучения, чтобы понимать, на каком этапе обучения я нахожусь
- Stu-01-004 : Я, как администратор автошколы, хочу изменять статус ученика, чтобы фиксировать завершение обучения или результат экзамена

### Training Context
- Tra-01-001 : Я, как администратор автошколы, хочу создавать план обучения для ученика, чтобы организовать процесс его подготовки
- Tra-01-002 : Я, как администратор автошколы, хочу добавлять теоретические и практические занятия в план обучения, чтобы сформировать расписание занятий
- Tra-01-003 : Я, как ученик, хочу просматривать своё расписание занятий, чтобы своевременно посещать обучение
- Tra-01-004 : Я, как ученик, хочу подтверждать предложенное расписание занятий, чтобы согласовать обучение с личным временем
- Tra-01-005 : Я, как администратор автошколы, хочу отмечать занятие как проведённое, чтобы фиксировать прогресс обучения
- Tra-01-006 : Я, как администратор автошколы, хочу отмечать план обучения как завершённый, чтобы допускать ученика к экзамену

### Exam Context
- Exm-01-001 : Я, как администратор автошколы, хочу создавать заявку на экзамен для ученика, завершившего обучение, чтобы передать его данные в ГАИ
- Exm-01-002 : Я, как администратор автошколы, хочу отмечать заявку как отправленную в ГАИ, чтобы контролировать процесс допуска к экзамену
- Exm-01-003 : Я, как сотрудник автошколы, хочу фиксировать результат экзамена ученика, чтобы сохранять информацию о сдаче экзамена
- Exm-01-004 : Я, как ученик, хочу просматривать результат экзамена, чтобы понимать, успешно ли был сдан экзамен

## Выводы

Работа была выполнена для изучения подхода Domain-Driven Design и освоения принципов выделения bounded contexts, агрегатов и связей между контекстами на основе предметной области. В процессе выполнения работы были приобретены навыки анализа предметной области, построения user stories, проектирования карты контекстов и моделирования системы с использованием основных концепций DDD.
